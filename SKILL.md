## ⚡ 分层动员机制（v2.0 核心升级）

v2.0 最大的架构升级：从"是否触发"二选一 → "需要动员多少力量"三级分层。

### 三级动员模式

```
用户输入
    ↓
┌─────────────────────────────────────────┐
│ 龙心OS · 始终在线                        │
│ 判断：任务类型？复杂度？需要哪些引擎？   │
└────────────────┬────────────────────────┘
                 │
     ┌───────────┼───────────┐
     ↓           ↓           ↓
┌─────────┐ ┌─────────┐ ┌─────────┐
│ 轻载模式 │ │ 标准模式 │ │ 深度模式 │
│ 1个引擎  │ │ 2-3引擎 │ │ 5引擎+   │
│ 不调用   │ │ 龙脑OS  │ │ 龙脑OS  │
│ 龙脑/爪  │ │ 核心层   │ │ 全层+   │
│         │ │         │ │ 龙爪OS  │
└─────────┘ └─────────┘ └─────────┘
```

### 详细动员规则

| 任务类型 | 动员模式 | 引擎数 | 龙脑OS | 龙爪OS | 典型场景 |
|---------|---------|:------:|:------:|:------:|---------|
| 单文件读写 | **轻载** | 1 | — | — | 写邮件、查资料 |
| 知识学习（单域） | **轻载** | 1 | — | — | 读一篇文章 |
| 知识学习（跨域·含五行） | **标准** | 2 | 核心层 | 凤脑OS | 跨领域理解 |
| 五行人格分析 | **标准** | 2-3 | 核心层 | 五行心理学OS | 人格测评 |
| 味藏经营管理 | **标准** | 2-3 | 核心层+扩展层 | 味藏总智能体 | 门店管理 |
| AIOS架构设计 | **深度** | 5 | 全层 | 全部 | 系统升级 |
| 多元跨域融合 | **深度** | 5 | 全部 | 全部 | 复杂决策 |
| 重大危机处理 | **深度** | 5 | 全部+扩展 | 全部 | 危急情况 |

### 动员级别自动判定

- **轻载模式**：单领域+单引擎任务（S0/S1/S2），只激活1个引擎，不调用龙脑/龙爪
- **标准模式**：单领域+跨域或双引擎任务（S3/S4/S6/S7/S8），激活2-3引擎，龙脑OS核心层
- **深度模式**：多引擎+跨多域+系统级任务（S5/S9），5引擎全开，龙脑OS全层+龙爪OS多子系统
