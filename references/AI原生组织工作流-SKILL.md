```
┌────────────────────────────────────┐

└────────────────────────────────────┘
```

## 🔗 与五象限的本能映射

本Skill的[AI]/[人]/[⇄]标注是**人机协同五象限**战略模型在执行层的具体化。

| 五象限标注模式 | 对应的任务分解 |
|---------------|--------------|
| **Q1·效率协作者** | 🟢 [AI] 执行指令 → 🟡 [⇄] 等待验收 |
| **Q2·知识拓展者** | 🟢 [AI] 系统输出 → 🔴 [人] 确认理解 |
| **Q3·思辨对话者** | 🟡 [⇄] 悟空给方向 → 🟢 [AI] 发散 → 🔴 [人] 筛选 |
| **Q4·协同探索者** | 🔴 [人] 输出经验 → 🟢 [AI] 整理 → 🔴 [人] 验收 |
| **Q5·未知共创者** | 🟡 [⇄] 共同探索 → 🔴 [人] 直觉判断 → 🟢 [AI] 展开 |

### 本能三问→标注模式速查

| 悟空掌握度 | AI掌握度 | 本能判断 | 标注模式 |
|-----------|---------|---------|---------|
| 高 | 高 | "这是Q1·效率协作者" | 🟢 [AI] 全部 |
| 低 | 高 | "这是Q2·知识拓展者" | 🟢 [AI] → 🔴 [人] |
| 中 | 中 | "这是Q3·思辨对话者" | 🟡 [⇄] → 🟢 [AI] → 🔴 [人] |

| 低 | 低 | "这是Q5·未知共创者" | 🟡 [⇄] 探索 → 🔴 [人] 直觉 |

> **每次任务先过本能三问，再进入标注流程——这就是龙心OS的协作本能。**

## 🎯 三分类标注体系

### 标注符号

| 符号 | 类别 | 定义 | 谁做 | 示例 |
|------|------|------|------|------|
| 🟢 [AI] | **AI并行** | 我可以独立完成，无需人工干预，可以多任务同时进行 | 龙龟神将 | 搜索、生成、计算、分析、读文件、写代码、上传 |
| 🔴 [人] | **人串行** | 必须由悟空亲自判断/决策/批准，不可跳过 | 悟空 | 战略方向、标准定义、最终拍板、质量审查、新领域探索 |
| 🟡 [⇄] | **串联瓶颈** | AI并行完成后必须等待人处理才能继续的节点 | **此处等待悟空** | 等待审批、等待确认方向、等待提供信息、等待决策 |

### 标注层级

根据任务复杂度，标注分为三级：

**L1-轻载**（简单任务：1-3步）
```
示例：搜索竞品信息
🟢 [AI] 搜索并整理竞品资料
```

**L2-标准**（中等任务：4-10步）
```
示例：撰写公众号文章
🟡 [⇄] 悟空确认选题方向
🟢 [AI] 搜索相关素材
🟢 [AI] 撰写初稿 → 存入草稿箱
🔴 [人] 悟空审核正文
🟢 [AI] 根据反馈修改
🔴 [人] 悟空最终确认发布
```

**L3-深度**（复杂任务：10+步，多子系统）
```
示例：新知识领域学习
🟡 [⇄] 悟空指定学习范围和深度
🟢 [AI] 搜索相关文献
🟢 [AI] 逐行分析文本
🟢 [AI] 建立知识文档
🟡 [⇄] 等待悟空确认文档结构
🟢 [AI] 补充知识图谱+标签+链接
🔴 [人] 悟空审查知识质量
🟢 [AI] 三库同步存储
```

## ⚙️ 工作流执行标准

### 阶段1：任务分析（每次任务开始）

收到悟空的任务后，在**执行前**先做分解：

```
步骤1：识别任务的"可并行部分" → 🟢 [AI]
步骤2：识别任务的"必须人做部分" → 🔴 [人]
步骤3：标注中间的串联节点 → 🟡 [⇄]
步骤4：检查悟空是否已经提供了所有需要的上下文
```

**输出格式**（放在任务回复开头或回复中）：

```markdown
## 🗺️ 任务分解
- 🟢 [AI] xxx → 我已完成
- 🟡 [⇄] 等待你确认xxx
- 🔴 [人] 需要你决定xxx
```

### 阶段2：并行执行

- 所有 🟢 [AI] 部分**同时执行**（能并发的尽量并发）
- 标注执行进度 🟢已完成 / ⏳进行中

### 阶段3：收敛报告

当遇到 🟡 [⇄] 串联瓶颈时，清晰告知悟空：
1. 已完成 🟢 [AI] 部分的结果
2. 等待悟空 🟡 [⇄] 做哪个决定
3. 提供决策所需的完整信息（决策地图）

### 阶段4：完成总结

任务完成后，输出串行瓶颈统计（可选，仅当悟空完整参与时）：

```markdown
## 📊 任务统计
| 指标 | 数值 |
|------|------|
| 🟢 AI并行步数 | X步 |
| 🔴 人串行步数 | Y步 |
| 🟡 串联瓶颈数 | Z处 |
| 瓶颈率 | Z/(X+Y+Z) × 100% |
| 建议 | 下次可尝试压缩 [具体建议] |
```

## 🔗 与龙心OS其他Skills的整合协议

### 与「自主进化闭环」整合

- 每次任务标注数据自动写入 `EVOLUTION.md` 的"本周瓶颈"条目
- 当瓶颈率持续上升时，自动触发 GAPS.md 中的"组织能力缺口"记录

### 与「人机协同五象限」整合

| 标注类别 | 五象限对应 | 说明 |
|---------|-----------|------|
| 🟢 [AI] | Q1 效率协作者 + Q2 知识拓展者 | 标准化执行和信息获取工作 |
| 🟡 [⇄] | Q3 思辨对话者 | 需要悟空与龙龟对话确认 |
| 🔴 [人] | Q4 协同探索者 + Q5 未知共创者 | 涉及战略判断和新领域探索 |

### 与「知识学习引擎」整合

- 标注数据中发现的**重复瓶颈**→ 自动总结为知识点 → 存入 Obsidian
- 成功压缩的瓶颈案例 → 作为「组织改进案例」存档

### 与「知行合一」整合

```
表示：完成一次完整任务对话
压缩：提炼本次任务中出现的瓶颈模式
泛化：将压缩后的模式应用到下次类似任务
```

## 📋 使用规范

### DO ✅
- 每次任务开始时自动执行任务分解
- 在回复中清晰标注 [AI]/[人]/[⇄]
- 遇到 🟡 时提供完整决策信息
- 完成时输出瓶颈统计（如有）
- 发现重复性瓶颈时，主动提出压缩方案

### DON'T ❌
- 不要在简单任务（1-2步）中过度标注，那会让信息噪音化
- 不要在标注中解释太多——标注的作用是"一目了然"，不是"写说明文"
- 不要因为标注而改变原有的工作输出——标注是附加层
- 不要擅自将 🔴 改为 🟢——"受托楔子"不可逾越

### 自动报警规则

当出现以下情况时，自动向悟空报告：

| 条件 | 动作 |
|------|------|
| 连续3次任务瓶颈率>50% | 提示"组织瓶颈异常，建议进行一次深度诊断" |
| 同一类型瓶颈出现5次以上 | 提示"XXX瓶颈重复出现，建议制定压缩方案" |
| 本周瓶颈率相比上周上升>20% | 提示"组织能力退化预警" |
