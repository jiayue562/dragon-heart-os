# AIOS 三层协作指南

> 龙心 OS（L1 调度层）→ 龙脑 OS（L2 思维模型层）→ 龙爪 OS（L3 专业领域执行层）

---

## 1. 架构总览

龙心 OS 不是孤岛。它是 AI OS 六层架构中的调度中枢，向下联动龙脑 OS（思维模型层）和龙爪 OS（专业领域执行层）。

```
┌─────────────────────────────────────────────────────────────────┐
│  L1 · 龙心 OS（调度层）                                          │
│  功能：感知上下文 → 识别场景 → 分层动员 → 路由引擎 → 反馈学习     │
│  组成：1 个调度中枢 + 5 个子智能体                                │
│  入口：SKILL.md + auto-activate.json + framework/               │
│  5引擎：知行合一/知识学习/人机协同五象限/象思维/五色光思维         │
├─────────────────────────────────────────────────────────────────┤
│  L2 · 龙脑 OS（思维模型层）                                      │
│  功能：提供 100+ 思维模型、麦肯锡工具、教员方法论                  │
│  核心层：MECE/金字塔原理/第一性原理/系统思维/逻辑层次              │
│  扩展层：决策矩阵/7S分析/SCAMPER法/PREP法                       │
│  路由：model_router.py                                           │
│  调用时机：龙心 OS 判断需要思维模型辅助时自动路由                  │
├─────────────────────────────────────────────────────────────────┤
│  L3 · 龙爪 OS（专业领域执行层）                                   │
│  功能：五行心理学OS、味藏总智能体、岗位智能体系列                  │
│  子系统：                                                         │
│  ├─ 五行心理学OS：凤心/凤脑/凤爪 → 人格分析诊断转化              │
│  ├─ 味藏总智能体：宪法层路由 → 7S店长/岗位智能体                 │
│  └─ 岗位智能体系列：店长/经理/料理长/创业单元等                  │
│  路由：skill_registry                                             │
│  调用时机：龙心 OS 判断需要专业领域执行时自动路由                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 调用链

每个用户输入经过龙心 OS 场景识别后，根据分层动员级别决定是否联动 AIOS 其他层级：

### 轻载模式（S0/S1/S2）

```
用户输入 → 场景识别 → 轻载判定
  → 龙心OS: 单引擎响应（人机协同/知识学习）
  → 龙脑OS: 不调用
  → 龙爪OS: 不调用
  → 输出
特点: 最快响应速度，最小开销
适用: 日常对话、简单任务、深度阅读
```

### 标准模式（S3/S4/S6/S7/S8）

```
用户输入 → 场景识别 → 标准判定
  → 龙心OS: 2-3引擎协同（如五色光+知行合一）
  → 龙脑OS: 核心层模型辅助（MECE/金字塔/第一性原理）
     ↓ model_router.py
  → 龙爪OS: 定向调用对应子系统（可选）
     ↓ skill_registry
  → 整合输出
特点: 结构化深度，核心思维模型辅助
适用: 创意创新、分析决策、会议引导、知识编译、修行文化
```

### 深度模式（S5/S9）

```
用户输入 → 场景识别 → 深度判定
  → 龙心OS: 全5引擎协同
  → 龙脑OS: 全层模型（核心+扩展）
     ↓ model_router.py
  → 龙爪OS: 全部子系统联动
     ↓ skill_registry
  → 全景整合输出
特点: 最高决策质量，全栈联动
适用: 重大决策、系统进化
```

## 4. 集成要点

### 4.1 龙心 OS 是唯一入口

- 所有任务必须先经过龙心 OS 场景识别
- 龙脑 OS 和龙爪 OS 是被路由层，不主动触发
- 避免多头调度引起的冲突

### 4.2 轻载模式不联动

- S0/S1/S2 场景保证最快响应速度
- 无论龙脑 OS 和龙爪 OS 是否可用，轻载模式都不调用
- 但场景识别始终在龙心 OS 完成

### 4.3 深度模式全量联动

- S5/S9 场景需要最高决策质量
- 龙心 OS 全5引擎 + 龙脑 OS 全层 + 龙爪 OS 全部子系统
- 输出前做全景整合，避免信息孤岛

### 4.4 路由跨层不跨域

- 龙心 OS → 龙脑 OS：通过 model_router.py
- 龙心 OS → 龙爪 OS：通过 skill_registry
- 不跨层直接调用其他层的内部组件

## 5. 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 龙脑 OS 模型路由失败 | 降级到无模型辅助，龙心 OS 独立完成 |
| 龙爪 OS 子系统不可用 | 跳过该子系统，不影响其他引擎 |
| 分层判定失败 | 默认降级到轻载模式 |
| AIOS 联动超时 | 使用已获得的输出，不做"等待超时" |
