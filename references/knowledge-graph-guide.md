# 知识图谱集成指南

> 双向链接 + 五行分类图谱 · v2.0 新增

---

## 核心理念

知识天生是网状的，不是线性的。一个知识孤岛，不是知识，是信息。

龙心 OS 知识图谱的目标：让每个引擎的输出成为其他引擎的输入。木生火、火生土、土生金、金生水、水生木——不是修辞，是架构。

### 跨引擎生克应用

```
场景：用户问"帮我分析一下这个市场趋势"
→ 龙心OS：场景识别 → S4 分析决策
→ 分层动员：标准模式
→ 引擎：五色光思维（金）+ 知行合一（土）
  → 金生水：决策分析 → 需要深度知识支撑
  → 自动追加知识学习（水）引擎
  → 水生木：深度积淀 → 触发象思维（木）创意类比
  → 输出：结构化分析+深度洞察+创意类比
```

## 3. 双向链接规范

### 3.1 SKILL.md frontmatter

```yaml
---
name: 龙心 OS
tags:
  - 五行/土
  - 调度中枢
  - AIOS/L1
linked_skills:
  - 龙脑 OS
  - 龙爪 OS
  - 知行合一
  - 知识学习
  - 人机协同五象限
  - 象思维
  - 五色光思维
---
```

### 3.2 references 目录链接

每条 reference 文档的头部标注双向链接：

```markdown
---
linked_from: SKILL.md
linked_to: [龙脑 OS, 龙爪 OS]
category: aios_collaboration
---
```

### 3.3 案例文档链接

每条案例记录场景→引擎→输出的链接关系：

```markdown
## 案例: [标题]
scene: S4
engine: 五色光 + 知行合一
output_type: 分析报告
linked_engines: [知识学习, 象思维]
knowledge_graph_tags: [金生水, 水生木]
```

## 4. 知识图谱同步

### 4.1 同步目标

| 目标 | 路径 | 频率 |
|------|------|------|
| Obsidian | D:\以观其妙书院知识库\观其妙书院\01-龙心 OS 核心系统 | 每次对话完成后 |
| OpcClaw | C:\Users\...\OpcClaw 知识库\01-龙心 OS 核心系统 | 每次对话完成后 |

### 4.2 同步内容

- 新生成的案例 → 结构化知识文档
- 跨引擎链接 → 双向链接标注
- 五行分类 → 标签系统
- 图谱更新 → 总索引刷新

### 4.3 同步方式

通过 `hooks/before_start.py::after_complete` 触发器自动同步，不需要手动操作。

## 5. 查询方式

知识图谱上的信息可以通过以下方式查询：

- **按引擎查**: "上次象思维输出了什么？" → 查询所有象思维相关的案例
- **按场景查**: "S4 场景的历史输出" → 查询所有 S4 场景的案例
- **按五行查**: "火行相关的所有输出" → 查询人机协同引擎的所有输出
- **按链接查**: "从金到水的知识流动" → 查询五色光→知识学习的映射案例

## 6. 注意事项

1. 知识图谱是生长型的，不是一次性构建的
2. 每次对话完成后自动补充图谱节点
3. 双向链接必须在两端都标注，不能只标一端
4. 五行分类是"倾向性"而非"绝对性"
5. 图谱信息同步到 Obsidian 后自动更新知识图谱可视化
